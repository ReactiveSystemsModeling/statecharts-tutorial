Exercise 4

Integrate generated traffic light statechart to Java UI

Statecharts define reactive behavior that interacts with the environment. In this exercise code is generated from the traffic light statechart. This code is then integrated with a graphical UI of the traffic light.

What is in the project

You will find additional files in this exercise project.

entry purpose
JRE System Library Ignore it. Java is used and this is an Eclipse Java project wich requires the library.
src This folder contains the hand written Java code along with the code generated from the statechart. You can take a look at it.
04-Run-UI.launch This is a launch configuration that you can use to start the UI application. Simply right-click and choosee "Run As > 04-Run-UI".
sc.sgen This is the configuration for the code generator. This is required to generate code.

Inspect the code generator model

Open the sc.sgen file. The content is explained by comments in the file.

Inspect the generated code

The source code is located in the folder src. You can find the generated state machine code in the Java package traffic.light.trafficlightctrl. Take a look at the interface definition ITrafficLightCtrlStatemachine.java to get familiar with the state machines API. TrafficLightCtrlStatemachine.java contains the implementation.

Integrate the state machine

The statemachine is not yet attached to the UI. Open the Java class TrafficLightDemo.java in src/traffic.light and follow the instructions that you'll find in the source code comments.

Run the UI

If you have implemented the state machines event listerners you can run the application. Select the file 04-Run-UI.launch in the project explorer. Right-click the entry and select "Run As > 04-Run-UI". This will launch the UI (see screenshot). You can observe how the traffic light cycles through the different phases. This UI includes elements like buttons which do not work yet but will be integrated in the next exercises.
